# Export Templates Directory
**Project Name:** Indian Payroll Management System

This document outlines the strict formats for files generated by the application for external integration. These templates represent exactly how the data outputs are constructed.

---

## 1. Banking Integration
### 1.1 NEFT / RTGS Corporate Salary Batch Template (`.csv`)
Used internally to upload to HDFC / ICICI corporate portals for disbursement.
*(Module Source: `Step4_BankFile.jsx` & `PayrollOpsTab.jsx` exports)*

**Separator:** Comma (`,`) | **Headers:** Not expected directly in payload, but documented.

| Position | Field Target | Description / Constraints |
| :--- | :--- | :--- |
| **Col 1** | Transaction Type | Must natively map (e.g. `HDFC` or `NEFT`). |
| **Col 2** | Target Account No | String. Must perfectly mirror Employee profile. |
| **Col 3** | Net Amount Payable | Formatted exactly as `0.00` float natively. |
| **Col 4** | Beneficiary Name | Employee Name. |
| **Col 5** | Execution Date | Strict format `YYYYMMDD`. |
| **Col 6** | Narrative | Pre-programmed as `SALARY` or `SALARY_[MMYYYY]`. |
| **Col 7** | IFSC Code | Crucial if transaction leaves primary host bank. |

**Output Example Preview:**
```csv
HDFC,501002345678,41500.00,ANIL KUMAR,20260428,SALARY,HDFC0001234
NEFT,111456999999,28500.00,SUNITA PRASAD,20260428,SALARY,SBIN0004561
```

## 2. Statutory Compliances
### 2.1 EPF ECR Return (`EPF_ECR_Upload.txt`)
Electronic Challan cum Return formatted strictly for the EPFO portal.
*(Module Source: `Step5_Statutory.jsx`)*

**Separator:** Custom strictly (`#‾#`) | **Extension:** `.txt`

| Position | Field Target | Description |
| :--- | :--- | :--- |
| **1** | UAN Number | 12 Digit Universal Account Number. |
| **2** | EPS Gross | Pensionable Salary Gross (Capped/Uncapped). |
| **3** | EPF EE Contribution | Employee 12% absolute value. |
| **4** | EPS Contribution | Employer 8.33% absolute value. |
| **5** | EPF ER Differential | Employer 12% - 8.33% (approx 3.67%). |

**Output Example Preview:**
```text
101112223333#‾#15000#‾#1800#‾#1250#‾#550
101569923444#‾#10000#‾#1200#‾#833#‾#367
```

### 2.2 ESI Return Format (`ESI_Return_Template.CSV`)
Standard ESIC compliance format.
*(Module Source: `Step5_Statutory.jsx`)*

**Separator:** Comma (`,`) | **Extension:** `.csv`

| Position | Field Target |
| :--- | :--- |
| **Col A** | Insurance Number |
| **Col B** | Attendance Days |
| **Col C** | Total Gross Salary |
| **Col D** | Reason Code (Only for 0 attendance) |
| **Col E** | Last Working Day |

**Output Example Preview:**
```csv
IP Number,Days,Gross,EE,ER
6712345600,30,18000,135,585
6712345601,28,15000,113,488
6712345602,0,0,On Leave,
```

## 3. Importer Modules Simulator
### 3.1 Initial Component Parsing Template (`.xlsx`)
Used by `Step0_SalaryBreakdown.jsx` to dynamically load external structures into memory.

| Header Row (Strictly required) | Type (Expected) | Field Constraints |
| :--- | :--- | :--- |
| **Name** | Custom String | Display name of the component |
| **Type** | Defined Enum Matching | Basic, HRA, Allowance, Variable, Reimbursement... |
| **Amount** | Number | Raw numeric scalar or 0. |

> [!CAUTION]
> The engine's parser handles partial case-insensitive matches for the `Type` column (e.g., "Medical Reimbursement" safely yields type `reimbursement`), ensuring UX fluidity across large generic `.xls` files.
